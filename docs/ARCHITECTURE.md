# Архитектура проекта

Проект реализован в соответствии с принципами **Clean Architecture**, разделяя ответственность на четыре основных слоя.

## Структура слоев (lib/src)

### 1. Domain (Ядро)
Содержит бизнес-объекты (модели) и интерфейсы репозиториев. Этот слой не зависит от других слоев.
- `models/`: Неизменяемые модели данных (`Freezed`).
- `repositories/`: Абстрактные интерфейсы для работы с данными.

### 2. Data (Инфраструктура)
Реализация интерфейсов из Domain слоя.
- `repositories/`: Конкретная реализация репозиториев (например, `ReportRepositoryImpl`).
- Взаимодействует с внешними API через `http`.

### 3. Application (Логика приложения)
Связующее звено между UI и Data.
- `providers/`: Riverpod провайдеры для управления состоянием и внедрения зависимостей.
- `utils/`: Вспомогательные инструменты (экспорт в Excel, интероп с Telegram).

### 4. Presentation (UI)
Слой отображения, зависящий от Application слоя.
- `screens/`: Основные экраны приложения.
- `widgets/`: Переиспользуемые компоненты.

## Управление состоянием (Riverpod)
- Используется последняя версия Riverpod с кодогенерацией.
- Провайдеры разделены по функциональным областям (тема, отчеты, данные инициализации).
- Обработка асинхронных данных выполняется через `AsyncValue`.

## Принципы разработки
- **SOLID & DRY**: Избегание дублирования и четкое разделение ответственности.
- **Null Safety**: Полная поддержка и отсутствие оператора `!`.
- **Immutability**: Все модели данных неизменяемы.
